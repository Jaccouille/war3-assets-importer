Warcraft 3 Model Importer

Imports custom assets (3D models, textures, sounds, and any other files) into a Warcraft III map file (.w3x / .w3m), and optionally creates unit definitions so the models are usable inside the World Editor and in-game.

Quick Start

  1. Click Open Map to load a .w3x or .w3m map.
  2. Click Import Models Folder to scan a folder containing the assets you want to import.
  3. Select the files you want to import using the checkboxes in the Assets tab.
  4. Configure options in the Import Configuration tab.
  5. Click Process and Save — the processed map is written to the Output path.

Assets Tab

Displays discovered assets in two selectable views:

  • Category View — files grouped by type (Models / Textures), organised into the same folder hierarchy they have on disk.
  • Folder View — the raw filesystem tree rooted at the chosen folder. Shows file/folder count and total size in parentheses.

Use checkboxes to select which files to import. Checking a folder node selects everything inside it. Clicking a file node previews the asset on the right.

Import Configuration Tab

Unit Creation

Create units Add a custom unit definition for each selected MDX model. Each unit appears as a new entry in the Object Editor (W3U file).
Place units on terrain Place the created units on the map terrain. Draw a rectangle on the map preview to define the placement area, or leave it empty to use the full camera-bounded map area.
Clear all units Remove all existing unit placements from the map before importing. Useful for re-importing without accumulating duplicates.
Clear existing assets Delete all previously imported assets from the map archive before adding the new selection. Use this to replace an old import without leaving stale files.
Keep filename only Store imported assets without their subfolder path (e.g. Infantry/soldier.mdx → soldier.mdx). Command-button textures (BTN*/DISBTN*) always use their canonical WC3 paths regardless of this setting.
Unit Scaling Scale factor applied to the model size of created units. 1.0 = original size, 2.0 = double size.

Unit Placement

Unit Origin ID Base unit type ID used as the template for created units (e.g. hfoo for Human Footman). The new units inherit their base stats from this type. Type to search existing unit definitions in the loaded map.
Unit Angle Facing direction of placed units in degrees. 0° = East   90° = North   180° = West   270° = South (default WC3 camera direction).
Spacing X / Y Horizontal and vertical gap between units on the placement grid, measured in pixels relative to the displayed map image. Smaller values pack units more tightly.
Placing Order Whether the placement grid fills left-to-right row by row, or top-to-bottom column by column.

Unit Naming

Auto-name from MDX filename Automatically set each unit's display name by converting its MDX filename. Choose a style with the Format dropdown.
Auto-assign BTN icon If a BTN* texture matching the MDX filename is found among selected assets, automatically assign it as the unit's command-card icon.
Format How the filename is converted into a display name:
  Space Separated (keep case): InfantryAmerican → Infantry American
  Space Separated: → Infantry american
  camelCase: → infantryAmerican
  snake_case: → infantry_american
  UPPER_CASE: → INFANTRY_AMERICAN
Preview Opens a table showing how every selected MDX filename will be formatted with the current settings.

Map Preview

The right-hand panel shows the map image and lets you define the unit placement area.

  • Left-click and drag to draw a rectangle on the map image.
  • The blue triangles show where units will be placed at the current spacing.
  • Yellow triangles show units already present in the map.
  • Click Clear to remove the drawn shape; units will then use the full map area.
  • The status indicator is green when all units fit, red when the area is too small (increase the shape size or raise the spacing values).
